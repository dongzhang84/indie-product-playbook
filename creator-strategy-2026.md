# 从零到卖课：一份 120 天的自我梳理

## 一、先把目标和前提说清楚

这不是一份宏大的商业计划书，而是一个单干者——物理 PhD 出身、转过 AI、创过业失败过、现在人在美国、还要带娃——给自己画的一张行动地图。前提是单干、不靠机构背书、走 creator economy 路线。不考虑去抱学校或大厂的大腿，也不考虑拉团队搞规模化。

最终想做到三件事：

第一，**X（推特）粉丝涨到 1 万**。不是为了虚荣，是因为没有影响力就没有卖课的资格，粉丝数在目前阶段是衡量影响力最直接的代理指标。

第二，**每三周上线一个最小成本的产品**。不是为了靠产品赚钱——独立开发者的失败率在 90% 以上，幸存者偏差不能当做路径依赖。做产品的真实目的是练手、攒 portfolio、给未来的卖课积累案例和素材，顺便——万一跑通了呢。

第三，**实现付费**。哪怕第一次只卖出三份，都比三个月后全新启动强。有了买家才有反馈、才有迭代素材、才有往下走的动力。

B2B 和咨询不在这个阶段的讨论里。美国的咨询市场早就过饱和，一个创业公司随便提个需求就能收到 800 封冷邮件，没有口碑和行业根基去切这块市场是空对空。长远可以再说，现在不碰。

贯穿这份计划的核心原则是一句话：**让机会选择你，不要自己预先把路堵死**。具体意思是，不预设"我只能教什么"、"我只能做什么"，而是广撒网、多试、看市场反馈，哪条线出信号就加码哪条。自媒体要发各种主题、产品要多做几个、卖课的形态要多试几种，都是这个逻辑。

## 二、三条主线怎么咬合

现在同时推进三条线：Vibe Coding（独立开发）、自媒体、卖课。这三条线表面是三件事，但在合适的 framing 下其实是一件事的三种形态。这个合流点是整份计划里最重要的发现，后面会单独展开。

**Vibe Coding 这条线的定位要想清楚**。如果把它定义为"做出爆款产品赚钱"，那就是跟 99% 失败率硬碰硬，没有意义。正确的定义是：把它当作技能训练 + 资产积累。每做一个产品，不管有没有人用，都产出了这些东西——一段可以放在 GitHub 上的代码、一次完整的产品开发经验、一个可以写进 X 的过程记录、一个可以拆解成卖课内容的真实案例。哪怕项目本身"失败"了，上面这些副产品也都还在。这样一来，独立开发的成功率对你就不再是 10%，而是 40-50%——因为你的"成功"标准变宽了。

**自媒体是现阶段唯一真正投入精力的主战场**。目标不是赚钱，是两件事：涨粉（积累将来卖课的影响力）和收信号（让市场告诉你哪类内容、哪类受众、哪种定位最有前途）。自媒体同时还逼着你持续输出，而持续输出就是持续学习——你为了写东西就必须去搞懂东西，这是副作用但很珍贵。

**卖课是中间环节，不是最终目标**。最终目标本来是 B2B，但现在不讨论了；退而求其次，卖课就是眼下能做的最实际的变现形式。但这里有一个必须正视的紧迫感——**AI 卖课的窗口期可能正在关闭**。2023 年 ChatGPT 刚出来时是黄金期，到 2025 年已经明显卷化，每半年门槛都高一些。等流量完全起来再启动卖课，很可能到时候窗口已经过了。所以不能按"先涨粉一年，再启动卖课"的串行逻辑来，必须并行。

## 三、平台选择：为什么 X 为主、知乎为辅

这是整个计划里讨论最久、也是最后才想清楚的一块。

可用的平台理论上很多：X、知乎、B 站、YouTube、小红书、公众号。但对一个单干 + 带娃 + 精力有限 + 人在美国的人来说，大部分都不值得碰。

**小红书直接排除**。女频属性太强（女性用户 85%+），科普和 AI 类内容流量天花板很低；而且它吃的是"人设+情绪价值+图文卡片"的玩法，跟理工背景的人天然不合，硬切等于折磨自己。

**公众号排除**。海外身份开号非常困难，绕过去的操作（反向注册之类）有合规风险，不值得为一个已经过了流量红利期的平台折腾身份问题。

**YouTube 排除**（至少现阶段）。视频制作成本是图文的 5-10 倍，精力切得太碎；而且中文 YouTube 的 AI 赛道流量远不如 B 站，冷启动周期太长。

**B 站保留一个轻量的选项**——旧号"祭天惊人"三年没更新了，但重启成本几乎为零，发两三条新东西看看平台给不给流量。给就认真做，不给就放回去。

**真正的主力是 X，副力是知乎**。

为什么这么选？因为这个选择直接对应了卖课的最终目标。

**知乎的用户画像是学生为主**。这些人求知欲旺盛、会点赞会收藏，但付费能力低、也不是会为 AI 课掏钱的那群人。知乎这 5 年前就有的问题是变现难——不是平台不给钱，是平台内变现天花板太低（做得好的头部月入 2-3 万封顶），真正赚钱的人都是把知乎当入口、把流量导到站外的私域（知识星球、公众号、Substack）去变现。你 5 年前在知乎流量不小但没能变现，根本原因就是没有搭站外承接。

**X 的用户画像则完全不同**。中文 X 圈虽然总量只有几十万活跃用户，但浓度极高——在职工程师、独立开发者、AI 从业者、想出海的人、海外华人。这些人有美元支付能力、有真实的付费意愿、并且是将来 B2B 的潜在客户池。一个 X 的垂直粉丝在商业价值上可能顶得上 10-50 个知乎粉丝。

所以结论很清楚：**X 为主，知乎作为副产品同步**。不是两边对称投入，而是所有内容先按 X 的逻辑来写，知乎同步发一份、能爆就爆、不爆也不调整方向去迎合。

## 四、X 的真实运作机制——别被表象骗了

在 X 上涨粉，第一件要破除的迷信是"只要内容好，算法会推，自然就涨粉"。这在英文 X 和中文微博上或许成立，**在中文 X 上基本不成立**。

中文 X 是一个熟人社会，不是一个广场。算法推来的冷流量——你看到一条推文有一万阅读但只多了 3 个粉丝——这不是你内容不好，是中文 X 的结构性问题。这里有一个名词可以精确描述这个现象：**read-follow gap**（读了但不关注的鸿沟）。读一条推文是 3 秒决策、低门槛；但关注一个账号是高门槛决策——它意味着"我预期这个人未来会持续给我价值"。冷流量几乎不会做这个高门槛决策。

那中文 X 涨粉的真实路径是什么？是圈子。

具体的链条是这样的：**你进入某个圈子 → 被圈子里的 node 账号注意到 → 被他们回复/转发/点赞 → 圈子里其他人发现你 → 关注你**。算法流量是"外人经过"，圈子流量是"被引荐"。后者的关注转化率可能是前者的 50-100 倍。

这就解释了一件你之前可能觉得困惑的事——**为什么中部账号（几千到几万粉）比头部大 V（几十万粉）对你更有价值？** 

头部大 V 的"不理你"不是针对你，是他们的默认状态。他们每天被 @ 几百次、被留言几百条，不可能回。他们的算法位置已经稳了，不需要通过"发现新人"来维护形象。头部账号之间也很少互推，怕被认为合作或背书。

而中部账号正在涨上去的路上，他们需要同盟、需要评论圈活跃、需要被 taste 好的人关注。一个几千到几万粉的中部 node，跟你这种 1600 粉但有独特背景的账号互动，对他也有收益。而且中部账号之间互相推、互相带人是日常——**圈子的裂变从中部开始，不是从顶部**。

所以涨粉这件事，每天花 30 分钟在 5-10 个目标中部账号下面留**有质量的评论**（不是"说得好""学到了"这种水评论，是带独立观点或补充信息的评论），比你多发 50 篇文章都有效。这是最重要的单一动作，其他所有发文、写推、做内容，都是在这个基础上才有意义。

## 五、中文 X 上的人群画像——以及那个扎心的矛盾

看清楚中文 X 上到底有哪些人很重要，因为这决定了你该跟谁互动、忽略谁、服务谁。

几个主要人群：

**第一类是国内翻墙出来的软件工程师**。主要诉求是扩大影响力、找赚钱机会、看海外动态。对情怀没兴趣，务实、直接。

**第二类是海外政治圈**。每天围绕共产党、美国大选、各种意识形态战争打转。人数庞大但跟你的商业目标完全不相关，忽略。

**第三类是做 AI/独立开发者的圈子**。这几年崛起的一群，做 side project、做 SaaS、聊 Vibe Coding、聊 indie hacking。代表账号比如 idoubi、小互、熠辉 Indie、宝玉。这个圈子核心活跃用户大概 1-2 万，浓度极高、互相关注、口碑传播快。**这就是你真正的主战场**。

**第四类是流量变现党**。贴搬运视频、追热点、蹭马斯克的流量分成。跟你的定位冲突，忽略。

**第五类是海外求职/润学圈**。留学生、H1B、想润的、已润的。关心签证、跳槽、薪资、海外生活。你的海外背景在这个圈子有发言权。

**第六类是 AI 行业从业者/研究者**。在大厂 AI 部门、在学术圈的华人。数量少但一个点赞或转发能带来大量曝光。值得建立弱连接。

**第七类是内容/自媒体创业者**。做 newsletter、做 YouTube、卖课的那群人。这个圈子提供方法论参考和可能的合作机会。

这样梳理下来，你的策略就清楚了：**主战场是第一类和第三类（程序员 + AI 独立开发者）**，战略投资第五、六、七类（保持连接），忽略第二、四类。

但这里藏着一个非常扎心的矛盾，必须直面：**X 粉丝最愿意互动的那群人（程序员），恰恰是最不爱付费的人群**。

这是 AI/技术圈的一个文化传统——GitHub 开源精神、"知识应该免费"、付费教程被视为 low、买书买订阅可以但买课抗拒。Cursor 订阅 $20/月没人心疼，但一个 $30 的 AI 编程课会被骂"割韭菜"。

这就意味着 X 上涨的粉，直接卖课给他们的转化率会低于预期。那 X 的粉丝对你的真实价值是什么？是**口碑扩散**（程序员圈口碑传得极快）、**同行协作机会**、**长期的 reputation 积累**。卖课的直接付费，可能更多要依靠另一类人群——下一节会讲。

## 六、卖课的两条路——以及两个真实案例

在 AI 卖课这件事上，中文圈目前有两条明显分叉的路径，代表人物各不相同。

**第一条是"水课 + 营销 + 焦虑贩卖"路线**。最极端的代表是李一舟——三年赚 1.75 亿，199 元入门课再引导升 1980 元高阶课，标榜清华博士人设，靠短视频投放和直播转化，内容浅薄但销量巨大。后果是小程序被封、面临侵权起诉、口碑全网崩坏。这条路能赚快钱，但赚完就完了——它跟你想要的长期 reputation 和 B2B 桥梁完全相反。

**第二条是"实操 + 订阅 + 持续沉淀"路线**。代表人物是熠辉 Indie（@yihui_indie），他的模式值得拆解。

熠辉的打法是这样的。人设直接写进 handle——"Indie"就是独立开发者，知乎自我介绍"开源和 AI 代码拼装工程师"，**主动降低学历感、强调实操**，走的不是"我是专家"路线，是"我也在一线折腾、只是手快一点"路线。

内容模式非常固定——**每次新工具一出来（Claude Code、MCP、n8n、spec-kit……），他立刻做两件事**：一是发短推分享使用技巧（利他、快、转发率高），二是做一个端到端能跑的完整小项目（比如"Claude Code + n8n 做小红书爬虫 + AI 仿写 Agent"）。他不讲原理、不做趋势分析、不写长篇思考——程序员圈最不爱看这些。他只讲"照着做就能跑"的东西。

变现模式是关键——**订阅制知识库，不是一次性大课**。早期他尝试过一次性卖课（AI 编程课），后来转向 FlowUs 付费知识库订阅（¥199/月或 ¥1999/年），每周更新一个新内容。这个模式的数学是：500 个年付 ¥2000 的用户就是 100 万流水，不需要大粉丝量，**核心是转化率和留存率**。

订阅制的好处一连串：决策门槛低（用户觉得随时能取消）、收入可复利（不像一次性课卖完就完）、持续更新解决内容过时问题（每周新工具节奏对应不住手痒的独立开发者）、交付的不是"知识"（这群人白嫖惯了）而是"现成能跑的项目包"（省时间 = 值钱）。

**你能从熠辉身上直接搬过来的，有这么几条**：

订阅制优于一次性大课——不求单次爆款，求持续付费。内容单元 = 端到端小项目包，不是讲义、不是教程、不是原理讲解。Handle 全平台统一，品牌一致性。**最重要的是——不要去包装"专家"身份**，反权威的"代码拼装工程师"人设反而好卖。你前面一直纠结"半路出身讲 AI 显得搞笑"，熠辉直接证明了这种担忧是多余的——不需要博士光环，需要的是一线实操的可信度。

对你来说两个差异要注意：他在国内，你在美国，FlowUs 这类平台对海外用户有摩擦，你要用 Substack、Gumroad、Patreon 做类似承接。以及他的内容极其工具化、跟得紧——这是体力活，你要做类似的事意味着每周都要追工具，预估好这个投入。

## 七、三条线合流——这份计划里最重要的发现

前面说三条主线（Vibe Coding、自媒体、卖课）是一件事的三种形态，这里具体展开。

熠辉的模式证明了一个路径：

**做一个 Vibe Coding 小项目 → 过程发到 X 上 → 成品放进订阅知识库 → 累积成课**。

这四步看起来是四件事，实际上是**一次投入、三次复用**——你花时间做了那个小项目（这是 Vibe Coding 线），过程中写下的每一步、每一次踩坑、每一次发现都是天然的 X 内容（这是自媒体线），完成后的项目包作为付费知识库的一期更新（这是卖课线）。

换句话说：你之前以为三条线要并行推进、精力要切成三份，实际上可以**共用同一份产出**。你不是在做 Vibe Coding 的同时也做自媒体也准备卖课——你是在做同一件事的三个不同展示面。

这个 framing 一旦想通，很多焦虑就不存在了。你不需要"抽时间"做内容，做项目的过程本身就是内容。你不需要"准备卖课素材"，每个项目就是一个素材。你也不需要担心"没有产品怎么卖课"——熠辉式的知识库就是你的产品。

这个合流点就是整份计划的发动机。后面所有的执行动作，都围绕这一个发动机来展开。

## 八、题材即受众——三篇文章的实测数据

在具体的执行方案之前，还有一个在实操中最容易犯错的地方要说清楚：**题材本身就决定了受众、决定了平台，不是包装和改写能解决的**。

这不是理论推演，是你自己三篇文章的实测数据得出的结论。

你写过一篇"为什么现在的大学生都不太想当科学家了"，用 1927 索尔维会议的合影开场，讲得很动人。知乎 331 赞、277 收藏、51 评论、1486 关注。X 上几乎没人看。

你写过一篇两万字的"黄仁勋芯片不是浓缩铀"，目录完整、叙事宏大（鹰派叙事、五层蛋糕、三条通道、双人舞、苏联剧本失效）。知乎 61 赞、1 人听过。X 上基本没人读。

你还写过一篇"AI 将如何颠覆教育"，讲李飞飞的观点、K-12、ChatGPT 作弊。X 上 5000 阅读（这是你在 X 的最高纪录），知乎 80+ 赞。

三篇文章清清楚楚地说明了一件事：**同一个写作者、同样的水平，不同题材在两个平台上的命运完全相反**。

知乎爆款题材（学生焦虑、科研困境、宏大历史叙事、人文思辨）在 X 上必死。原因不是 X 不推长文、不是你首句不够抓人、不是算法歧视——是 **X 用户不想看这些**。他们是在职工程师、AI 从业者、独立开发者，他们关心的是能用的工具、能做的项目、能赚的钱、能看懂的行业动态。你讲"人类潜力的浪费"，他们无感。

反过来，你那篇 AI 教育能在 X 上跑到 5000 阅读，**因为 AI 教育是 X 用户关心的题材**——里面有 ChatGPT、有行业趋势、有对从业者的启示。这类内容在知乎反而冷淡，因为知乎主力学生不关心宏观行业走势，他们关心"我该不该读博"。

所以**标题本身就是内容的一部分，题材本身就是对受众的筛选**。"大学生不想当科学家"这个标题就注定了读者是学生和关心学生的人；"AI 颠覆教育"这个标题就注定了读者是关心 AI 的人。你不可能用一套内容通吃两边。

这带来一个必须接受的现实——**你要涨的 X 粉，大概率不是喜欢你知乎爆款内容的那群人**。知乎给你的点赞越多，反而越是一种警告：这个题材方向长远来看跟你的卖课目标是错配的。你最成功的那篇"物理人不想当科学家"如果真的让你成为了"物理转行议题博主"，那你将来要怎么卖 AI 课？你的受众根本对付费学 AI 没兴趣。

**所以得做一个痛的切割**——知乎的反馈是真实的成就感，但不能让它主导写作方向。题材要按 X 来选，不按知乎来选。

## 九、X 长文的生态变化——别用老经验

关于 X 上长文的态度，这里也要做一次校准。

过去几年的经验是"X 上越短越好，长文没人看"。但 2025-2026 年这个规则已经变了。X 算法在持续弱化语言和社交关系边界、强化内容表现力指标（停留时间、互动率、跨圈层扩散）。长文天然获得更长停留时间，在新算法下**反而有优势**。Article 功能（Premium 专享）给的权重也比普通推文高。

所以在中文 X 上确实能看到博主的 article 跑出万级甚至十万级阅读——这不是错觉，是真实现象。

但几个限定条件必须说清楚：

第一，**X 长文冷启动比知乎难得多**。知乎有问题作为天然流量入口（回答热门问题自带流量），X 没有，所有内容从零开始。1600 粉的账号发两万字，哪怕内容再好，冷启动也极难。你的"黄仁勋"X 上没人看，既有题材错配的问题，也有杠杆不够的问题。

第二，**X 长文和知乎长文是两种写法**，虽然都是长文。X 长文要先抛结论再展开，知乎长文习惯先铺背景再推论；X 长文要有金句开头，知乎长文可以放目录；X 长文讲具体立场，知乎长文讲宏大叙事。文字可以通用，结构不能通用。

第三，**长文和短推并行，不是二选一**。长文做深度沉淀和算法权重，短推做高频曝光和圈子互动。只做长文的账号粉丝粘性好但增速慢，只做短推的账号流量快但缺乏代表作。两者结合才是完整的。

对你的意义：继续写长文，但题材按 X 来选、结构按 X 来学。同时每天发一些短推维持存在感，拆解长文做内容复用，这些都是基本功。

## 十、执行方案：接下来 100 到 120 天做什么

前面聊的所有东西，最后落到具体动作上是这些。时间框架是 100-120 天，大概对应 3 到 4 个月。

**主战场是 X，知乎作为同步副产品**。这句话决定了所有内容的写法——按 X 的口味和题材写，发完同步到知乎，知乎能爆是 bonus、不爆不调整方向。

**题材锁定两条主线——AI 实操 + 海外观察**。AI 实操是 Vibe Coding 过程中的踩坑、新工具的评测、完整项目的复盘。海外观察是人在美国的信息差——美国 AI 圈的动态、硅谷公司在做什么、招聘市场、行业八卦、政策走向。这两类都是 X 用户真正想看的东西。

**放弃宏大叙事类作为主力**。"大学生不想当科学家"、"物理学已死"、"AI 会颠覆人类"这类题材偶尔写一篇过瘾可以，不能当主力。这些题材在知乎反馈很好，但跟卖课目标错配，越写越偏。

**节奏：每周 5 篇，100 天大约 70-85 篇**。AI 辅助写作可以跑到这个量——你负责框架和判断，AI 负责扩写和润色。核心不是产量，是方向不漂移。

**Vibe Coding 每三周一个小产品，同一个产品直接产出多条内容**。一个产品的开发过程可以拆成：起手动机（一条推）、选型决策（一条）、踩坑记录（2-3 条）、完成 demo（一条）、复盘总结（一条长推或 article）。这样三周一个产品对应 6-8 条 X 内容，不是另外再挤时间做内容。

**handle 和 bio 必须改到 3 秒能看懂**。类似"物理 PhD 转 AI / 在美国 / 独立开发中"这样的配置，让路过的人立刻知道你是谁、关注你能得到什么。现在这个阶段，profile 优化是最便宜也最被低估的涨粉动作。

**每天 30 分钟做圈子互动，这个比发内容还重要**。在 5-10 个目标中部账号下留有质量的评论——熠辉 Indie、Mr. Panda 那个层级的人。别去追头部，头部不理任何人。中部是圈子裂变的起点，也是你现在能挤进去的那一层。

**搭一个站外承接点**。不要让 X 和知乎的流量白白流走。一个 Substack 邮件订阅、或一个 Patreon 页面、或一个简单的落地页——任何能收集将来买家信息的地方都行。每篇内容末尾带一个低摩擦的钩子（"想继续看这类内容，订阅我的 newsletter"）。

**100-120 天内不一定要上付费产品，但要想清楚第一个最小付费产品长什么样**。可能是一个 $19 的 PDF、一个 $29 的短指南、一个 $49 的 Notion 模板——重点是低价、轻量、这个周末能做完。它的作用不是赚钱，是在 profile 上挂出"这个人有东西卖"的信号，跑通从内容到付费的链条。

**120 天结束时要能回答这几个问题**：X 粉丝涨了多少（目标是从 1600 到至少 3000-4000，为后续冲万粉打基础）、哪类内容反馈最好、Vibe Coding 做了几个产品、哪个产品引起了真实兴趣、有没有跑通第一笔付费、还是完全没动静。

**硬性 gate**：如果 120 天后 X 粉丝涨幅 < 500、没有任何内容获得真正的圈内互动、Vibe Coding 产品没做出一个像样的、也没卖出任何一份哪怕 $19 的东西——这个信号说明方向有根本问题，需要重新审视。不是小修小补，是重新定位。

但也要诚实地管理预期：**Read-follow gap 是结构性的，冷启动就是慢的**。如果你 4 个月涨了 1500 粉、Vibe Coding 做出了 4-5 个小项目、有 20-30 个付费用户、在熠辉或 Mr. Panda 这个层级的中部账号圈子里有了存在感——这已经是非常健康的进展，哪怕离 1 万粉还很远。

## 十一、最后一件事

这份计划里最不需要反复讨论的东西，也是过去聊得最多的东西——**定位**。

你擅长什么、你是不是物理 PhD、该不该强调学历、半路出身讲 AI 会不会搞笑——这些焦虑本质上是用理论推演代替市场反馈。没有人知道真正的答案在哪，也没有人能在脑子里推出来。你要做的不是在开始前把定位想清楚，是**广撒网、多产出、看数据、让观众告诉你你是谁**。

熠辉 Indie 的定位不是他在脑子里想出来的，是他做了两年内容、读者反复跟他聊、看哪类内容反馈好之后慢慢沉淀下来的。Mr. Panda 的定位也是。AB Kuai.Dong 的定位更是——10 年积累出来的。

所以现在别再纠结"我应该教什么、我是谁"这类问题了。**开始发内容、开始做产品、开始互动**。100-120 天之后再回头看，那时候定位会自己浮现出来，而且比你现在推演的任何版本都准确。

这份计划不是蓝图，是一份允许自己迭代、允许改变、允许被打脸的行动提纲。唯一不能变的只有一件事——**要做事、要发东西、要露脸**。想得再清楚，不发出去就是零。
