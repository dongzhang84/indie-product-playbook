# 02 - 逆向工程

> 不是照抄代码，而是拆解产品逻辑、用户流程、MVP边界。

## 🎯 目标

**从"这个产品不错"到"我知道怎么做出来"**

### 不是什么

```
❌ 不是看竞品的源代码
❌ 不是1:1复制所有功能
❌ 不是研究他们的技术架构

这些都不重要，而且浪费时间
```

### 是什么

```
✅ 理解核心价值：解决什么问题
✅ 拆解用户流程：用户怎么使用
✅ 确定MVP边界：最少做哪些功能
✅ 找到差异化：你能做得不同的地方
```

## 📋 逆向工程Checklist

### Step 1：试用产品（30分钟）

**作为真实用户使用：**

- [ ] 注册账号（如果需要）
- [ ] 完成核心任务1次
- [ ] 尝试所有主要功能
- [ ] 记录使用过程的感受

**记录：**
- 哪个功能最有用？
- 哪个功能可有可无？
- 什么地方卡住了？
- 什么地方体验好？

### Step 2：拆解用户流程（30分钟）

**画出用户旅程：**

```
Landing Page
    ↓
    [看到什么？被什么吸引？]
    ↓
Sign Up / Start Using
    ↓
    [需要注册吗？还是直接用？]
    ↓
Core Action
    ↓
    [核心功能是什么？]
    ↓
Result
    ↓
    [用户得到什么？]
    ↓
Conversion/Retention
    [为什么付费？为什么回来？]
```

**记录每一步：**
- 需要输入什么？
- 系统做了什么？
- 用户看到什么？

### Step 3：功能清单（30分钟）

**列出所有功能，分类：**

#### 核心功能（Must Have）
```
没有这些功能，产品就不成立
例子：
- AI图片生成器 → 生成图片
- 时间追踪器 → 记录时间
```

#### 重要功能（Should Have）
```
提升体验，但不是第一天必须的
例子：
- 历史记录
- 导出功能
- 基础设置
```

#### 锦上添花（Nice to Have）
```
产品有用户后才考虑
例子：
- 高级分析
- 团队协作
- API
```

#### 完全不需要（Don't Need）
```
对MVP无用的功能
例子：
- 多语言（除非产品本身就是国际化的）
- 复杂的权限系统
- 社交分享（可以后加）
```

### Step 4：技术拆解（30分钟）

**确定技术实现：**

- [ ] 前端需要什么：
  - 几个页面？
  - 需要什么UI组件？
  - 有复杂交互吗？

- [ ] 后端需要什么：
  - 需要数据库吗？
  - 需要什么API？
  - 需要定时任务吗？

- [ ] 第三方服务：
  - AI API（Claude/GPT）
  - 支付（Stripe）
  - 认证（Supabase Auth）
  - 其他？

### Step 5：MVP定义（15分钟）

**你的MVP包含：**

```
Week 1目标：
1. 核心功能能用
2. 一个简单的landing page
3. 能注册/登录（如果需要）
4. 基础的UI（不用完美）

不包含：
❌ 完美的设计
❌ 所有边缘case
❌ 复杂的设置
❌ 付费功能（可以先标记"Coming Soon"）
```

### Step 6：差异化（15分钟）

**你的产品和原版的区别：**

可能的差异化：
- [ ] 更便宜
- [ ] 更简单（功能少但专注）
- [ ] 更快
- [ ] 更好的UI
- [ ] 面向不同用户群
- [ ] 增加1个独特功能

**记住：不需要完全不同，1-2个小差异就够了。**

## 🎪 实际案例

### 案例：假设要山寨Time Ledger

#### Step 1：试用

```
发现：
- 核心功能：tap记录时间，看成本
- 很简单，没有复杂设置
- iOS only，没有web版
- 免费，无付费功能
```

#### Step 2：用户流程

```
1. 打开app
2. 看到今天花了多少时间/钱
3. Tap一个活动（工作、娱乐等）
4. 开始记录
5. 完成后，tap停止
6. 看到总结（今天花了多少时间在什么上）
```

#### Step 3：功能清单

**核心功能：**
- 开始/停止计时
- 显示当前活动
- 显示今天总时间
- 时间→金额换算

**重要功能：**
- 预设活动列表
- 历史记录
- 简单统计

**Nice to Have：**
- 详细分析
- 导出报表
- 多设备同步

#### Step 4：技术实现

```
前端：
- Next.js web app（他是iOS，你做web）
- 2个主要页面：
  • 首页（计时器）
  • 历史页面

后端：
- Supabase（存储记录）
- 简单的表结构：
  • users
  • activities（活动类型）
  • time_logs（时间记录）

第三方：
- 不需要AI
- 不需要支付（先做免费版）
- Supabase Auth就够了
```

#### Step 5：MVP定义

```
Week 1 MVP：
✅ 用户可以注册/登录
✅ 可以开始/停止计时
✅ 可以选择活动类型
✅ 可以看今天的总时间
✅ 简单的landing page

不做：
❌ 复杂的统计
❌ 导出功能
❌ 多语言
❌ 移动app（只做web）
```

#### Step 6：差异化

```
你的差异化：
1. Web版（原版是iOS only）
   → Android用户可以用

2. 免费 + 付费tier
   → 免费：基础功能
   → 付费：高级统计+导出（$5/月）

3. 团队功能（可选）
   → 小团队可以一起track时间
```

## ⚠️ 常见错误

### 错误1：想一次做完所有功能

```
❌ "我要把他们的所有功能都做出来"
✅ "我只做核心的3个功能"
```

### 错误2：过度创新

```
❌ "我要加10个新功能让它更好"
✅ "我先做和他们一样的，加1个小差异"
```

### 错误3：纠结技术细节

```
❌ 研究他们用什么技术栈
✅ 用你熟悉的栈做出来就行
```

### 错误4：完美主义

```
❌ "UI要和他们一样精美"
✅ "UI能用就行，先验证需求"
```
