# Twitter Following Summarizer - Chrome Extension Proposal

## One-liner

> "每天早上打开Chrome，自动帮你读完30-50个关注账号的推文，生成一份你真正想看的日报。"

---

## 背景与痛点

Twitter的following timeline本质上是一个**信噪比极低**的地方：
- 算法随机插入内容，不按时间排序
- 每次refresh显示的内容都不一样
- 真正有价值的内容被大量噪音淹没
- 每天花大量时间刷timeline，收益却很低

**核心问题：** 你关注的人发了什么，你根本不知道有没有看全。

---

## 产品定位

- **用户：** 自己（MVP阶段）
- **核心价值：** 把30-50个关注账号每天的推文，经AI过滤和总结，生成一份分类日报
- **关注内容分类：**
  - **Indie / Build in Public**：新产品发布、收入milestone、经验分享
  - **政治：** 重大事件、观点交锋

---

## 技术方案

### 为什么选Chrome Extension

| 方案 | 优点 | 缺点 |
|------|------|------|
| Twitter API | 稳定、可靠 | Basic版$100/月，成本高 |
| 服务器脚本 | 可定时自动跑 | 需要维护服务器 |
| **Chrome Extension** | **免费、无需服务器、用自己的登录session** | **Chrome需要开着** |

### 核心架构

```
[账号名单] → [逐个访问主页] → [抓取24小时推文] → [LLM总结] → [插件内展示]
```

**关键选择：不从timeline抓，而是逐个访问账号主页**
- 原因：Timeline受算法干扰，内容不完整
- 账号主页按时间排序，保证不漏

### 数据存储

使用Chrome内置的`chrome.storage.local`：
- 账号名单
- 每天抓取的推文内容
- 历史总结记录
- 无需外部数据库，数据完全本地

### 定时触发

使用`chrome.alarms` API实现每日定时任务。

**限制：** Chrome需要在后台运行。建议设定每天早上9点自动触发，起床开电脑后10-15分钟内总结完毕。

---

## MVP功能范围

### 做（Week 1）

- **账号管理页面：** 手动输入30-50个Twitter用户名，保存到本地
- **定时抓取：** 每天定时逐个访问账号主页，抓取过去24小时推文
- **AI总结：** 调用Claude/GPT API，按分类生成摘要
- **结果展示：** 点开插件图标即可查看当天日报

### 不做（后续再说）

- 自动import following列表
- 推送通知（邮件、Telegram等）
- 历史数据分析
- 多用户支持
- 移动端

---

## AI总结策略

**关键原则：先过滤，再总结，而不是直接总结**

### 两步走

**第一步：打标签 + 评分**
```
给每条推文打标签：
- 类别：indie / 政治 / 噪音
- 价值评分：1-10分
- 判断依据：是否有实质内容（数据、观点、事件）
```

**第二步：只对高分内容做摘要**
```
Indie类关注：
- 新产品发布 / launch
- 收入或用户milestone
- 有价值的经验/教训分享

政治类关注：
- 重大政策或事件
- 有影响力的观点交锋
- 值得追踪的进展
```

## 开发计划

### Week 1：MVP

| 时间 | 任务 |
|------|------|
| Day 1-2 | Chrome Extension基础框架、账号管理页面、`chrome.storage`存储 |
| Day 3-4 | 浏览器自动化：逐个访问账号主页、DOM抓取推文内容 |
| Day 5-6 | LLM API集成、两步总结逻辑（打标签→摘要） |
| Day 7 | 插件UI打磨、定时任务配置、本地测试 |

### 之后（视情况）

- 批量导入following列表（从Twitter数据存档）
- 账号分级：核心20人每天看，其余每周看一次
- 优化总结质量（prompt调优）
- 添加关键词过滤

## 技术栈

| 模块 | 技术 |
|------|------|
| Extension框架 | Chrome Extension Manifest V3 |
| UI | HTML + Vanilla JS（保持简单） |
| 数据存储 | `chrome.storage.local` |
| 网页抓取 | Content Script + DOM操作 |
| 定时任务 | `chrome.alarms` |
| AI总结 | Claude API 或 OpenAI API |

---

## 成本估算

**开发成本：** $0（自己做）

**运行成本（每天）：**
- 45个账号 × 平均5条推文 = 约225条
- 两步LLM调用：约$0.05-0.15/天
- 每月约：$1.5-4.5

**结论：** 几乎免费的个人工具。

---

## 风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| Twitter改版DOM，抓取失效 | 中 | 需手动更新插件，属于维护成本而非封号风险 |
| 被Twitter封号 | 低 | 只读不写，行为接近正常浏览 |
| LLM总结质量差 | 中 | 先过滤后总结策略 + 持续调优prompt |
| Chrome没开，定时任务不跑 | 中 | 接受此限制，养成早上开Chrome的习惯 |

先给自己用，做到真正好用，之后再考虑是否分享给别人。
